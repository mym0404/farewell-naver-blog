ExportOptionsPanel derives block option cards from getNaverBlogEditorBlockOutputDefinitions(), then groups the registry array by editor with groupBlockOutputDefinitionsByEditor(). The rendered groups appear only when more than one editor group exists, and the underlying registry already filters selectable entries to outputOptions with at least two variants. The new UI test asserts editor group order naver-se4, naver-se3, naver-se2 and card order naver-se4:formula/code/table/image/divider, naver-se3:table/code/image, naver-se2:table/divider/code/image, so duplicate blocks remain visible instead of block-type deduped. No UI path imports blockOutputFamilyDefinitions directly.
